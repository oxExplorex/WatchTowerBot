Менеджер Telegram-аккаунтов для личного использования.

## Возможности
- Управление Telegram-сессиями:
  - добавление аккаунтов
  - включение/выключение сессий
  - удаление сессий
- Отслеживание изменений диалогов:
  - новые чаты
  - удалённые чаты
  - опционально: боты и каналы
- Получение временных вложений (TTL/спойлер-медиа) в админ-бот
- ИИ-ответы через Gemini:
  - команда через префикс `.` (например: `.привет`)
  - поддержка текста, фото, видео, video note (кружок), voice/audio
  - учитывает контекст текущего диалога
- Статистика стабильности сессий и проверок (по часам/дням)

## Технический стек
- Python: `3.14.x`
- Telegram Bot API: `aiogram 3.x`
- User-аккаунты Telegram: `pyrofork`
- ИИ: `google-generativeai` (Gemini)
- База данных: `PostgreSQL` (локальная рекомендуется)
- ORM: `SQLModel` + `SQLAlchemy Async`

## База данных
При запуске бот автоматически:
- создаёт БД (если отсутствует)
- создаёт таблицы
- добавляет недостающие колонки (базовая авто-миграция схемы)

Можно использовать удалённую PostgreSQL БД, но это не рекомендуется из-за рисков утечки данных/сессий.

## Запуск
1. Скопировать `data/config_sample.py` в `data/config.py`.
2. Заполнить параметры бота, Gemini и PostgreSQL.
3. Установить зависимости из `requirements.txt`.
4. Запускать через `start.bat`.
5. Обновлять через `update.bat`.

## Обновление проекта
`update_bot.py` обновляет проект из ветки `main` и не перезаписывает локальные чувствительные данные:
- `data/config.py`
- `data/proxy.txt`
- `data/session/`
- `data/logs/`
- `data/temp/`

## ⚠️ Use At Your Own Risk
- Использовать только в личных целях и только с личными Telegram-аккаунтами.
- Возможна блокировка Telegram-аккаунта из-за частых/постоянных запросов.
- Использование полностью на ваш страх и риск.
- Не рекомендуется запускать на сторонних системах (VDS/VPS).
- Рекомендуется запуск только на личном устройстве и с локальной PostgreSQL БД, чтобы снизить риск утечки сессий и данных.
