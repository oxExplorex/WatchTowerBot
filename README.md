> ⚠️ Возможна блокировка Telegram-аккаунта при частых запросах и автоматизации.

> Без этого Gemini-функции не будут работать.


> 🚫 Продажа доступа к боту недопустима.
> Проект бесплатный, не содержит коммерческих функций и не предназначен для перепродажи услуг.

## Что умеет бот
- Управление Telegram-сессиями: добавление, включение/выключение, удаление.
- Мониторинг чатов: новые, удалённые, отдельно боты/каналы.
- Пересылка временных вложений (сообщения с одним просмотром):
  - фото и видео пересылаются в чат уведомлений.
- Ответы через Gemini по префиксу `.` с учетом контекста.
- Статистика состояния аккаунтов.
- Настройки: автообновление, часовой пояс, перезапуск.
- Уведомляет о новых версиях.

## Ограничения
- Userbot работает в режиме прослушивания и не имеет буферизации, как Telegram Bot API.
- При потере соединения на устройстве, где запущен скрипт, часть сообщений может быть пропущена.

## Настройка конфигурации
Проект читает переменные окружения из файла `.env` в корне.

1. Скопируйте `.env.example` в `.env`.
2. Заполните токены/ключи и параметры БД.
3. Установите зависимости: `pip install -r requirements.txt`.

Основные переменные:
- `TOKEN_BOT`
- `GEMINI_KEY`
- `ADMIN_ID_LIST` (через запятую)
- `DB_ENGINE`, `DB_HOST`, `DB_PORT`, `DB_USER`, `DB_PASSWORD`, `DB_NAME`, `DB_ADMIN_NAME`

## Запуск
Windows:
- `start.bat` — запуск бота
- `update.bat` — обновление проекта и зависимостей

Ubuntu/Linux:
- `start.sh` — запуск бота
- `update.sh` — обновление проекта и зависимостей

Если запуск успешный, в логах появятся строки:
- `Starting bot bootstrap`
- `Bot was started`

## База данных
При старте бот автоматически:
- создает БД (если отсутствует),
- создает таблицы,
- применяет совместимые авто-миграции (новые поля/типы).

Рекомендуется локальная PostgreSQL.

## Прокси для Gemini
Прокси привязывается к владельцу аккаунтов (`admin_id`):
- один владелец -> один прокси,
- все его Telegram-аккаунты используют этот же прокси.

Поддерживается только HTTP.

Форматы ввода:
- `ip:port:user:password`
- `ip:port@user:password`

Если прокси перестает отвечать:
- Gemini автоматически отключается для этого владельца,
- отправляется уведомление о проблеме.

## Обновление
`scripts/update_bot.py` скачивает актуальный репозиторий (ветка `main`) и не перезаписывает чувствительные локальные данные:
- `.env`
- `.env.local`
- `data/session/`
- `data/logs/`
- `data/temp/`

## Перенос на второй ПК
1. Установить Python 3.14 и PostgreSQL (или подключиться к существующей БД).
2. Клонировать репозиторий.
3. Создать `.env` из `.env.example`.
4. Заполнить токены/ключи/параметры БД.
5. Установить зависимости: `pip install -r requirements.txt`.
6. Запустить `start.bat` (Windows) или `start.sh` (Linux).

Если переносите существующие аккаунты-сессии:
- скопируйте папку `data/session/` на второй ПК,
- укажите корректные `DB_*` в `.env`.

## Важно
- Используйте на личном устройстве.
- Не рекомендуется запуск на сторонних VDS/VPS.
